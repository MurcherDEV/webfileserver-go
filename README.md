# NexusDrive (Personal File Server)

![NexusDrive Logo](https://img.shields.io/badge/Status-Active-success) ![Go Version](https://img.shields.io/badge/Go-1.22-blue) ![Docker](https://img.shields.io/badge/Docker-Ready-2496ED)

NexusDrive — это легковесный, безопасный и энергоэффективный персональный файловый сервер, оптимизированный для работы на домашних серверах и мини-ПК (Ubuntu Server 24.04). 

Система предоставляет красивый веб-интерфейс для управления файлами и поддерживает протокол **WebDAV** для монтирования хранилища как сетевого диска прямо в операционной системе.

## 🌟 Особенности
- **Единый бинарный файл**: И фронтенд, и бэкенд упакованы в один легковесный Docker-контейнер (около 20 МБ) благодаря интеграции через `//go:embed`.
- **Без тяжелых фреймворков**: Фронтенд написан на чистом Vanilla JS (с Fetch API для асинхронных операций), HTML5 и CSS3.
- **Premium Дизайн**: Эффект Glassmorphism (матовое стекло), переключение тем (Темная/Светлая), плавающие градиенты на фоне и плавные анимации.
- **Автоматический SSL (HTTPS)**: Встроенная интеграция с Nginx и Certbot для автоматического получения и продления сертификатов Let's Encrypt.
- **Изоляция данных**: Хранение загруженных файлов в безопасном Docker Volume `file_data`.

## 📋 Требования (Prerequisites)

Перед развертыванием убедитесь, что на вашем сервере (Ubuntu 24.04):
1. Установлен **Docker** и современный плагин **Docker Compose v2** (`docker compose` без дефиса).
2. Открыты порты **80** и **443** (в UFW или роутере).
3. Домен (например, `murcher.ru`) ссылается на публичный IP-адрес вашего сервера (A-запись в DNS-панели регистратора).

## 🚀 Пошаговое руководство по развертыванию

### Шаг 1. Перенос файлов на сервер
Скопируйте все файлы проекта на ваш Ubuntu Server. Например, в директорию `/opt/nexusdrive`.

```bash
# Перейдите в директорию проекта на сервере
cd /opt/nexusdrive
```

### Шаг 2. Инициализация SSL-сертификатов Let's Encrypt
Для того чтобы HTTPS заработал, необходимо получить SSL-сертификат от Let's Encrypt. В проекте есть скрипт, который автоматизирует этот процесс (создает "заглушку" Nginx, подтверждает владение доменом и выпускает сертификат).

Сделайте скрипт исполняемым и запустите его:
```bash
chmod +x init-letsencrypt.sh
sudo ./init-letsencrypt.sh
```
*Скрипт запросит подтверждение. Если вы разворачиваете проект впервые, ответьте `y`. Дождитесь надписи "Reloading nginx".*

### Шаг 3. Запуск инфраструктуры
После успешного получения сертификатов запустите всю связку контейнеров в фоновом режиме:

```bash
sudo docker compose up -d
```

Docker соберет образ Go-приложения (скачает зависимости, скомпилирует бинарник и упакует фронтенд) и поднимет Nginx и Certbot.

### Шаг 4. Проверка статуса
Убедитесь, что все 3 контейнера запущены и работают без ошибок:
```bash
sudo docker compose ps
```

## 🔐 Смена пароля (Важно для безопасности!)

По умолчанию сервер использует:
- **Логин:** `admin`
- **Пароль:** `admin`

Чтобы сменить эти данные, вам больше не нужно лезть в код приложения!

1. Откройте файл `docker-compose.yml`:
   ```bash
   nano docker-compose.yml
   ```
2. Найдите переменные `AUTH_USER` и `AUTH_PASS` в секции `environment` сервиса `app`:
   ```yaml
       environment:
         - AUTH_USER=your_new_username
         - AUTH_PASS=your_super_secret_password
   ```
3. Пересоберите и перезапустите бэкенд, чтобы изменения вступили в силу:
   ```bash
   sudo docker compose up -d --build app
   ```
*(Бэкенд сам безопасно захэширует ваш пароль алгоритмом bcrypt при запуске).*

## 💻 Использование системы

### 1. Веб-интерфейс
Откройте в браузере ваш домен:
👉 **https://murcher.ru**

Вы увидите экран логина. Введите ваши учетные данные. Через веб-интерфейс вы можете:
- Загружать файлы и папки.
- Скачивать файлы одним кликом.
- Удалять данные.
- Перемещаться по директориям.

### 2. Подключение через WebDAV
WebDAV позволяет подключить ваше облако к компьютеру так, будто это обычная папка на жестком диске или флешка.

**Для Windows:**
1. Откройте «Этот компьютер» (My Computer).
2. Нажмите правой кнопкой мыши по пустой области -> «Подключить сетевой диск» (Map network drive).
3. Выберите любую свободную букву диска.
4. В поле «Папка» введите: `https://murcher.ru/webdav/`
5. Поставьте галочку «Использовать другие учетные данные» (Connect using different credentials).
6. Введите ваш логин и пароль.

**Для macOS:**
1. В Finder в верхнем меню выберите Переход (Go) -> Подключение к серверу (Connect to Server) или нажмите `Cmd + K`.
2. Введите: `https://murcher.ru/webdav/`
3. Нажмите Подключиться и введите логин и пароль.

**Для мобильных устройств (iOS/Android):**
Рекомендуем использовать файловые менеджеры с поддержкой WebDAV, такие как **Documents by Readdle** (iOS) или **Cx File Explorer** (Android), добавив новый хост: `https://murcher.ru/webdav/`.
